# BlackBox Frontend

React приложение для управления и просмотра конфигураций устройств.

## Структура

Приложение состоит из трех основных вкладок:

1. **Девайсы** - отображает уникальные конфиги с расширением и время их создания. Поддерживает сортировку по времени создания.

2. **Версии** - отображает все версии конфигов с названием, временем изменения и временем создания. Поддерживает сортировку по времени изменения.

3. **Изменения** - место для side-by-side сравнения версий конфигов (в разработке).

## Установка и запуск

### Локальная разработка

```bash
cd frontend
npm install
npm start
```

Приложение будет доступно по адресу `http://localhost:3000`

### Переменные окружения

Создайте файл `.env` в папке `frontend`:

```
REACT_APP_API_URL=http://localhost:8080
```

### Docker

```bash
# Запуск через основной docker-compose.yml (рекомендуется)
docker-compose up -d --build

# Или отдельно frontend
docker-compose -f docker-compose.frontend.yml up -d --build
```

## Решение проблем

### Network Error при загрузке данных

Если вы видите ошибку "Network Error", проверьте:

1. **API сервер запущен**: Убедитесь, что сервис `api-web` запущен и доступен на порту 8080
   ```bash
   docker ps | grep api-web
   curl http://localhost:8080/health
   ```

2. **Правильный URL API**: По умолчанию используется `http://localhost:8080`. Если API доступен по другому адресу, создайте файл `.env` в корне проекта:
   ```
   REACT_APP_API_URL=http://your-api-url:8080
   ```
   Затем пересоберите frontend:
   ```bash
   docker-compose build frontend
   docker-compose up -d frontend
   ```

3. **CORS**: API сервер должен разрешать запросы с frontend. В текущей конфигурации CORS настроен для всех источников (`*`).

4. **Порты**: Убедитесь, что порты 3000 (frontend) и 8080 (API) не заняты другими приложениями.
